# Work Item 107: Fix deterministic agent color assignment

## Problem
GP-6 requires deterministic agent identity — color derived from agent type. `get_or_create_agent` in `WorldStateManager` hardcodes `color="white"` for all new agents. When `AgentInferenceEngine` updates an agent's type, it doesn't update the color.

## Fix

### world_state/manager.py
```python
from hamlet.inference.types import TYPE_COLORS, AgentType as InfAgentType

# In get_or_create_agent, when creating a new agent:
initial_type = AgentType.GENERAL
initial_color = TYPE_COLORS.get(InfAgentType.GENERAL, "white")
agent = Agent(..., inferred_type=initial_type, color=initial_color, ...)
```

### inference/engine.py
In `_handle_post_tool_use`, when updating agent type:
```python
from hamlet.inference.types import TYPE_COLORS

color = TYPE_COLORS.get(inferred, "white")
await self._world_state.update_agent(
    agent_id,
    inferred_type=ws_type,
    color=color,
)
```

## Notes
- `TYPE_COLORS` maps inference `AgentType` to color strings — verify it exists in `inference/types.py` (it should after work item 099)
- The initial color for GENERAL should match the entry in TYPE_COLORS, not be hardcoded
- After this fix, different agent types will show different colors in the TUI (matches GP-6)
