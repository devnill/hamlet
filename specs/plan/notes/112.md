# Work Item 112: Config validation in Settings.load()

## Problem
`Settings.load()` accepts any valid JSON without checking whether values are sensible. A user who sets `mcp_port: "banana"` gets a confusing error at runtime, not at startup. GP-11 (Low-Friction Setup) requires helpful error messages that guide users to fixes.

## Implementation
Add a `_validate()` method to `Settings`:

```python
def _validate(self) -> None:
    if not isinstance(self.db_path, str) or not self.db_path:
        raise ValueError(f"db_path must be a non-empty string, got: {self.db_path!r}")
    if not isinstance(self.mcp_port, int) or not (1 <= self.mcp_port <= 65535):
        raise ValueError(f"mcp_port must be between 1 and 65535, got: {self.mcp_port!r}")
    if not isinstance(self.activity_model, str) or not self.activity_model:
        raise ValueError(f"activity_model must be a non-empty string, got: {self.activity_model!r}")
    # Add tick_rate if it exists in Settings
    if hasattr(self, "tick_rate"):
        if not isinstance(self.tick_rate, (int, float)) or self.tick_rate <= 0:
            raise ValueError(f"tick_rate must be positive, got: {self.tick_rate!r}")
```

Call `self._validate()` at the end of `load()` before returning, and also after constructing defaults.
