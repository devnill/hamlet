# WI-159: Fix active_tools counter decrement in inference engine

## Context
`_handle_pre_tool_use` increments `session.active_tools += 1` on every PreToolUse event. `_handle_post_tool_use` does not decrement it. After the first tool call completes, `active_tools` is permanently ≥ 1. Every subsequent PreToolUse passes the `session.active_tools > 0` check in `_detect_spawn` and triggers a new agent spawn.

Also, `state.pending_tools` is populated in `_handle_pre_tool_use` but never consumed, growing without bound.

## Fix

### src/hamlet/inference/engine.py — _handle_post_tool_use

After identifying the session, decrement active_tools:
```python
session.active_tools = max(0, session.active_tools - 1)
```

Remove the pending_tools entry. The key used in `_handle_pre_tool_use` is typically `event.tool_name` or `event.id`. Read the current code to determine the exact key used, then evict it:
```python
state.pending_tools.pop(event.tool_name, None)  # or whatever key is used
```

## Notes
- Read engine.py before modifying — focus on `_handle_pre_tool_use` (where active_tools is incremented and pending_tools is populated) and `_handle_post_tool_use` (where the decrement should happen)
- Verify what key is used for pending_tools (tool_name vs event.id vs session_id)
- The decrement guard `max(0, ...)` prevents underflow if PostToolUse arrives without a preceding PreToolUse
