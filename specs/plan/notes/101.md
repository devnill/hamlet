# Work Item 101: Fix PersistenceFacade constructor call in __main__.py

## Problem
`__main__.py` calls `PersistenceFacade(db_path=settings.db_path)` but `PersistenceFacade.__init__` expects `config: PersistenceConfig | None = None`. This raises `TypeError` at startup.

## Fix
```python
from hamlet.persistence.types import PersistenceConfig

config = PersistenceConfig(db_path=settings.db_path)
persistence = PersistenceFacade(config=config)
```

## Notes
Read `src/hamlet/persistence/facade.py` to confirm the exact constructor signature before applying the fix. If `PersistenceConfig` has additional fields that map to `Settings` attributes (e.g., `event_log_max_entries`), wire those too.
