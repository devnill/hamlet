# WI-122: Consume notification_message and stop_reason downstream

## Context

WI-120 added `notification_message` and `stop_reason` as typed fields to `InternalEvent` and populated them in `event_processor.py`. The extraction layer is complete, but no downstream consumer reads these fields:

1. `inference/engine.py _handle_notification()` is an explicit no-op placeholder (`"""Handle Notification events — placeholder."""` with no body). It does not update `last_seen` for agents in the session.
2. `inference/engine.py _handle_stop()` updates `last_seen` for session agents but ignores `event.stop_reason`.
3. `world_state/manager.py handle_event()` builds the event log summary as `f"{event.hook_type.value}: {event.tool_name or ''}"`. For both Notification (tool_name=None) and Stop (tool_name=None) events, this produces `"Notification: "` and `"Stop: "` with no semantic content.

## Fix

### inference/engine.py — _handle_notification()

Implement minimally:
```python
async def _handle_notification(self, event: InternalEvent) -> None:
    """Handle Notification events — update agent last_seen and log message."""
    session = self._state.sessions.get(event.session_id)
    if session:
        for agent_id in session.agent_ids:
            self._state.last_seen[agent_id] = event.received_at
    if event.notification_message:
        logger.debug("Notification [session=%s]: %s", event.session_id, event.notification_message)
```

### inference/engine.py — _handle_stop()

Add stop_reason logging (existing logic: update last_seen — keep it):
```python
async def _handle_stop(self, event: InternalEvent) -> None:
    """Handle Stop events — update last_seen and log stop reason."""
    session = self._state.sessions.get(event.session_id)
    if session:
        for agent_id in session.agent_ids:
            self._state.last_seen[agent_id] = event.received_at
    if event.stop_reason:
        logger.debug("Stop [session=%s, reason=%s]", event.session_id, event.stop_reason)
```

### world_state/manager.py — handle_event() summary

Change the summary construction to include semantic content:
```python
# Before:
summary=f"{event.hook_type.value}: {event.tool_name or ''}",

# After:
if event.hook_type.value == "Notification" and event.notification_message:
    summary = f"Notification: {event.notification_message}"
elif event.hook_type.value == "Stop" and event.stop_reason:
    summary = f"Stop: {event.stop_reason}"
else:
    summary = f"{event.hook_type.value}: {event.tool_name or ''}"
```

## Key files

- `src/hamlet/inference/engine.py` — _handle_notification(), _handle_stop()
- `src/hamlet/world_state/manager.py` — handle_event() summary field

## Scope note

This work item implements minimum viable consumption: update last_seen and log. It does NOT implement full visual handling (e.g., showing notification messages in the TUI, or transitioning agent states differently based on stop_reason). That is future work. The goal here is to ensure the extracted data flows at least one level further than the InternalEvent boundary.
