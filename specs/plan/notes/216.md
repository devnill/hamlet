# WI-216 Implementation Notes

## Context

Guiding Principle 5: "Villages can expand, connect via roads, and found new settlements."

`ExpansionManager.process_expansion` (expansion.py:66-86):
1. Calls `check_village_expansion` — finds a suitable site at 20-50 cells distance
2. Calls `create_road_between` — draws road structures to the site
3. **Never creates a village at the site**

The expansion site is computed and the road is drawn, but no new `Village` record is ever created.

## Fix

### `WorldStateManager` — add `found_village` method

Add a method to create a new village for an existing project at a given position:

```python
async def found_village(self, project_id: str, center: Position, name: str) -> Village:
    """Create a new village at center for project_id and persist it."""
```

This should:
1. Acquire `self._lock`
2. Check if a village already exists very close to `center` (idempotency guard — avoid duplicates from concurrent ticks)
3. Create a new `Village` record with a generated ID, `project_id`, `center`, `name`
4. Insert into `_state.villages` and `_state.village_by_project` (or use a multi-village lookup)
5. Queue a DB INSERT via persistence facade
6. Return the new `Village`

Note: `_state.village_by_project` is currently a `dict[str, Village]` (one village per project). With multi-village support, this may need to become `dict[str, list[Village]]`. Alternatively, `found_village` can use the grid and `_state.villages` list directly, bypassing `village_by_project` for now.

### `ExpansionManager.process_expansion` — call `found_village` after road

After `create_road_between`, call:
```python
if hasattr(world_state, "found_village"):
    village_name = f"{village.name} Outpost"
    await world_state.found_village(village.project_id, site, village_name)
```

Using `hasattr` guard is consistent with the existing `create_structure` guard in the same file.

### Idempotency

On the next tick, `check_village_expansion` runs again. The new village appears in `all_villages`, so `_is_clear_site` will return `False` for sites within 15 cells of the new settlement. This naturally prevents re-founding the same site.
