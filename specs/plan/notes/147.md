# WI-147: Fix StructureUpdater dict unpacking bug

## Context
`src/hamlet/simulation/structure_updater.py` calls:
```python
await world_state.update_structure(structure.id, {"stage": new_stage, ...})
```
But `update_structure` in manager.py expects `**fields` (keyword args), not a positional dict.
The dict is treated as a single unknown field name and skipped, so structures never progress.

## Fix
Change the call to use keyword args:
```python
await world_state.update_structure(
    structure.id,
    stage=new_stage,
    material=new_material,
    work_units=0,
)
```
