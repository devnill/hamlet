# WI-161: Create initial village structures on village founding

## Context
`add_work_units()` in `manager.py:511-521` finds the nearest structure of the given type and silently returns when none exists. Nothing proactively creates initial structures (LIBRARY, WORKSHOP, FORGE) when a village is founded.

On a fresh world, villages have zero structures, so every work unit from every tool call is dropped. Structure-building mechanic (GP-8) is permanently broken on a new world.

## Fix

### src/hamlet/world_state/manager.py — get_or_create_village (or equivalent founding method)

When a NEW village is created (not loaded from persistence), seed it with initial structures:

```python
# After creating the village record, create initial structures
initial_structure_types = [StructureType.LIBRARY, StructureType.WORKSHOP, StructureType.FORGE]
for i, structure_type in enumerate(initial_structure_types):
    # Place near village center with slight offsets
    offset_x = (i - 1) * 3  # -3, 0, 3 relative to center
    position = Position(x=village.center.x + offset_x, y=village.center.y + 1)
    self._create_structure(village_id=village.id, structure_type=structure_type, position=position)
```

The `_create_structure` (or equivalent) method already exists and is called from `expansion.py:106`.

## Notes
- Read manager.py to understand how villages are created and what `_create_structure` (or `create_structure`) looks like
- Read expansion.py to see how `create_structure` is called — replicate that pattern
- The key constraint: only new villages get initial structures; villages loaded from persistence via StateLoader should NOT get duplicate structures
- Check if there's a flag or way to distinguish "just created" from "loaded from DB"
- Initial structures should have stage=0, material="wood" (earliest stage)
