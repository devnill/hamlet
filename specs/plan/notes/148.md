# WI-148: Fix StateLoader datetime string/object mismatch

## Context
`src/hamlet/persistence/loader.py` returns ISO format strings for timestamp fields (last_seen, created_at, updated_at), but WorldStateManager assigns these directly to Agent/Structure dataclass fields which expect datetime objects.

This causes type errors when zombie/idle detection performs datetime arithmetic (comparisons, total_seconds()).

## Fix in src/hamlet/persistence/loader.py
Where timestamp columns are read from DB, parse them:
```python
from datetime import datetime, timezone

def _parse_dt(value: str | None) -> datetime:
    if value is None:
        return datetime.now(timezone.utc)
    return datetime.fromisoformat(value)
```

Apply to all datetime fields in _load_agents, _load_structures, _load_projects, etc.

- Check that the DB stores timestamps as ISO strings (standard for SQLite)
