# WI-196: Functional gap — Bash tool_output schema widening and notification_type extraction

## Context

**Q-10**: The Bash tool returns `tool_output` as a plain string. `EVENT_SCHEMA` constrains `tool_output` to `["object", "null"]`, so Bash PostToolUse events are silently rejected by schema validation. Every Bash-heavy agent generates no world-state changes.

**Q-16**: `notification.py` sends `notification_type` in params. `validation.py` declares it in the schema. But `InternalEvent` has no `notification_type` field and `event_processor.py` does not extract it. The value is silently discarded.

## Changes

### src/hamlet/mcp_server/validation.py
Find the `tool_output` property in EVENT_SCHEMA. Change its type from `["object", "null"]` to `["object", "string", "null"]`.

### src/hamlet/event_processing/internal_event.py
1. Change `tool_output: dict[str, Any] | None` to `tool_output: dict[str, Any] | str | None`
2. Add field: `notification_type: str | None = None`

### src/hamlet/event_processing/event_processor.py
1. The existing `tool_output=raw.get("tool_output")` extraction already passes through any type. Verify this is correct — no `isinstance(val, dict)` filter should exist. If a filter exists, remove it.
2. Add `notification_type=raw.get("notification_type")` to the `InternalEvent(...)` constructor call.

### tests/test_event_processor.py
Add two tests:
1. `test_tool_output_string`: Send raw event with `tool_output="stdout content"`. Assert `event.tool_output == "stdout content"` (not None, not discarded).
2. `test_notification_type_extraction`: Send raw Notification event with `notification_type="progress"`. Assert `event.notification_type == "progress"`.

## Ordering

This item depends on WI-187 because WI-187 also modifies `tests/test_event_processor.py`. Apply WI-187 changes first, then add the two new tests here without conflicting with them.

The WI-187 parametrized tests do not assert on `tool_output` or `notification_type` specifically, so there is no logical conflict — just the file modification ordering.
