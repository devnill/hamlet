# WI-151: Add POST /hamlet/event HTTP endpoint to daemon

## Context
Hook scripts (hooks/pre_tool_use.py etc.) need to send events to the daemon via HTTP. The daemon currently only has GET /hamlet/state and GET /hamlet/events endpoints. There is no endpoint to receive incoming events from hooks.

The MCP server in mcp/server.py is a separate Claude Code plugin server (stdio transport) for hamlet_init tool — not for receiving hook events.

## Changes

### src/hamlet/mcp_server/server.py
Add POST /hamlet/event route:
```python
async def _handle_event(self, request: web.Request) -> web.Response:
    try:
        payload = await request.json()
    except Exception:
        return web.Response(status=400, text="Invalid JSON")
    if self._event_queue is not None:
        await self._event_queue.put(payload)
    return web.Response(status=200, text="ok")
```
Register: `app.router.add_post("/hamlet/event", self._handle_event)`

### hooks/pre_tool_use.py, post_tool_use.py, notification.py, stop.py
Each hook should POST JSON to http://localhost:8080/hamlet/event:
```python
import sys, json, urllib.request

def send_event(payload):
    try:
        data = json.dumps(payload).encode()
        req = urllib.request.Request(
            "http://localhost:8080/hamlet/event",
            data=data,
            headers={"Content-Type": "application/json"},
            method="POST",
        )
        urllib.request.urlopen(req, timeout=2)
    except Exception:
        pass  # graceful degradation — daemon may not be running

hook_data = json.loads(sys.stdin.read())
send_event({"hook_type": "PreToolUse", **hook_data})
```
