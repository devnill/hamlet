# WI-166 Notes: Add async:true to plugin hooks.json

## File to modify
`hooks/hooks.json`

## What to change
Add `"async": true` to each of the four hook command objects. The current structure is:
```json
{
  "type": "command",
  "command": "${CLAUDE_PLUGIN_ROOT}/hooks/pre_tool_use.py",
  "timeout": 5
}
```

After change:
```json
{
  "type": "command",
  "command": "${CLAUDE_PLUGIN_ROOT}/hooks/pre_tool_use.py",
  "timeout": 5,
  "async": true
}
```

Apply to all four hooks: PreToolUse, PostToolUse, Notification, Stop.

## Why async
The Python scripts already have a 1-second HTTP timeout internally and exit cleanly. Making hooks async means Claude Code does not wait for the hook to complete before proceeding, eliminating any latency impact on tool calls. This matches the behavior of the now-removed inline hooks in settings.json which were also async.
