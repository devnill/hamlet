# WI-176 Notes: Improve mcp import check in start.sh

## Context
The current check `python3 -c "import mcp"` only verifies the top-level mcp namespace is importable. A broken install with just the namespace directory would pass. Changing to `from mcp.server import Server` verifies the specific class used by `mcp/server.py` is available.

## File to modify
`/Users/dan/code/hamlet/mcp/start.sh`

## Current check (line 11)
```sh
if ! python3 -c "import mcp" 2>/dev/null; then
    echo "hamlet: mcp package not found. Install uv (https://docs.astral.sh/uv/) or run: python3 -m pip install mcp" >&2
    exit 1
fi
```

## Desired check
```sh
if ! python3 -c "from mcp.server import Server" 2>/dev/null; then
    echo "hamlet: mcp package not found. Install uv (https://docs.astral.sh/uv/) or run: python3 -m pip install mcp" >&2
    exit 1
fi
```

Only the import expression changes. The diagnostic message and exit code are unchanged.
