# WI-242: Threshold-Based Terrain Classification

## Context

With fBm producing a unified heightmap, we need to classify terrain types from elevation and moisture values instead of independent noise maps.

## Implementation Notes

### Location
- `src/hamlet/world_state/terrain.py` — replace `_generate_with_noise()` classification logic

### Classification Logic

The unified approach uses two noise layers:

1. **Elevation** (primary): Generated by `_warped_fbm()`
2. **Moisture** (secondary): Simple fBm with different seed

Classification rules (from research):

```python
def _classify_terrain(self, elevation: float, moisture: float) -> TerrainType:
    # Water at lowest elevations
    if elevation < self._config.water_threshold:
        return TerrainType.WATER

    # Mountains at highest elevations
    if elevation > self._config.mountain_threshold:
        return TerrainType.MOUNTAIN

    # Forest prefers higher moisture
    if moisture > self._config.forest_threshold:
        return TerrainType.FOREST

    # Meadow in moderate moisture
    if moisture > self._config.meadow_threshold:
        return TerrainType.MEADOW

    # Plains in dry areas
    return TerrainType.PLAIN
```

### Threshold Parameters

Existing thresholds are repurposed:
- `water_threshold: float = -0.3` — elevation below this = water
- `mountain_threshold: float = 0.7` — elevation above this = mountain
- `forest_threshold: float = 0.4` — moisture above this = forest (in passable areas)
- `meadow_threshold: float = 0.6` — moisture above this = meadow (in passable areas)

The `water_frequency` and `mountain_frequency` parameters become unused after this change. Keep them for backward compatibility but they won't affect generation.

### Visual Verification

After implementation, the terrain should show:
- Connected water bodies (lakes, not scattered ponds)
- Mountain formations (not isolated peaks)
- Forest clustering in moisture-rich areas
- Natural-looking transitions between terrain types
