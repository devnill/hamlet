# WI-157: Fix LegendOverlay visible reactive and toggle mechanism

## Context
WI-142 fixed HelpOverlay by removing its `visible: reactive[bool]` (which shadows DOMNode.visible) and using `overlay.display` instead. The same bug exists in LegendOverlay:
- `src/hamlet/tui/legend.py` still has `visible: reactive[bool] = reactive(False)`
- `action_toggle_legend` in app.py still uses `overlay.visible = not overlay.visible` instead of `overlay.display = not overlay.display`

## Changes

### src/hamlet/tui/legend.py
Remove:
- `visible: reactive[bool] = reactive(False)`
- `watch_visible()` method (if present)

Add/keep:
- `DEFAULT_CSS = "LegendOverlay { display: none; }"` (ensures hidden on startup)

### src/hamlet/tui/app.py
Change action_toggle_legend:
```python
def action_toggle_legend(self) -> None:
    try:
        overlay = self.query_one(LegendOverlay)
        overlay.display = not overlay.display
    except Exception as exc:
        logger.debug("toggle_legend: %s", exc)
```

### tests/test_tui_legend.py
Update tests that check `overlay.visible` to check `overlay.display` instead.
Remove tests that exercise the visible reactive if they exist.
