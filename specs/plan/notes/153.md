# WI-153: Implement work unit accumulation in PostToolUse handler

## Context
`inference/engine.py` PostToolUse handler infers agent type but does not add work units to structures based on duration_ms. This means structures never receive work units and never progress through stages. Violates GP-8 (Agent-Driven World Building).

## Existing Code
- `src/hamlet/inference/engine.py` — _handle_post_tool_use method
- `src/hamlet/world_state/manager.py` — add_work_units method (look for it)
- `src/hamlet/simulation/config.py` — work_unit_scale config value

## Fix in inference/engine.py
In _handle_post_tool_use, after type inference, add work units:
```python
# Get active structure for this agent's village
work_units = event.duration_ms * self._config.work_unit_scale
# Find agent's current structure and add work units
# (may need to query agent's village, then active structure)
await self._world_state.add_work_units(structure_id, work_units)
```

## Notes
- Read _handle_post_tool_use in engine.py first to understand current structure
- Read add_work_units in manager.py to understand its signature
- The agent → village → structure relationship needs to be traversed
- If no clear current structure, accumulate on the most recently touched structure for the village
- SimulationConfig.work_unit_scale should be 0.01 (1 unit per 100ms) per architecture spec
