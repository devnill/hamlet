# WI-246: Forest Clustering Algorithm

## Context

Forest from moisture threshold alone can still appear scattered. A seeding and growth algorithm creates more natural forest groves.

## Implementation Notes

### Location
- `src/hamlet/world_state/terrain.py` — add `generate_forest_groves()` function

### Seeding and Growth Algorithm

```python
def generate_forest_groves(
    self,
    grid: dict[Position, TerrainType],
    moisture_map: dict[Position, float],
    grove_count: int = 10,
    growth_iterations: int = 5
) -> dict[Position, TerrainType]:
    """Create forest groves from seeds in high-moisture areas."""
    result = grid.copy()

    # Find high-moisture passable positions for seeding
    high_moisture = [
        pos for pos, m in moisture_map.items()
        if m > self._config.forest_threshold
        and pos in grid
        and grid[pos] not in (TerrainType.WATER, TerrainType.MOUNTAIN)
    ]

    if not high_moisture:
        return result

    # Select N random seed positions
    self._random.seed(self._seed + 7777)
    seeds = self._random.sample(high_moisture, min(grove_count, len(high_moisture)))

    # Initialize groves
    groves: list[set[Position]] = [{seed} for seed in seeds]

    # Growth iterations
    for _ in range(growth_iterations):
        new_groves = []
        for grove in groves:
            # Expand each grove by adding passable neighbors
            expansion = set()
            for pos in grove:
                for dx, dy in [(-1, 0), (1, 0), (0, -1), (0, 1)]:
                    neighbor = Position(pos.x + dx, pos.y + dy)
                    if neighbor in grid and grid[neighbor] not in (TerrainType.WATER, TerrainType.MOUNTAIN, TerrainType.FOREST):
                        # Only expand if moisture is reasonable for forest
                        if neighbor in moisture_map and moisture_map[neighbor] > self._config.forest_threshold * 0.5:
                            expansion.add(neighbor)
            new_groves.append(grove | expansion)
        groves = new_groves

    # Apply forest to grid
    for grove in groves:
        for pos in grove:
            result[pos] = TerrainType.FOREST

    return result
```

```python
if self._config.forest_grove_count > 0:

    )
```

### Config Parameters

Add to TerrainConfig:
- `forest_grove_count: int = 10` — number of forest grove seeds
- `forest_growth_iterations: int = 5` — growth iterations per grove

### Relationship to Other Features

Forest clustering runs AFTER ridge generation and lake expansion. This ensures forests don't overwrite mountains or water. Forests are limited to passable terrain (not already water/mountain).
