# WI-208 Implementation Notes: Reset agent state to ZOMBIE on daemon startup

## Problem

`WorldStateManager.load_from_persistence()` restores agents with their stored database state:

```python
# src/hamlet/world_state/manager.py ~line 116
state=AgentState(d.get("state", "active")),
```

The inference engine's zombie detection (`_check_zombie`) only evaluates agents in its own
`_state.last_seen` dict. This dict is empty at startup. So any agents loaded from the DB
that were "active" or "idle" are never transitioned to zombie — they remain "active"
indefinitely, appearing on screen as spinning agents even when no Claude sessions are running.

## Fix

Change the single line in `load_from_persistence()` that sets agent state:

```python
# Before
state=AgentState(d.get("state", "active")),

# After
state=AgentState.ZOMBIE,  # always start as zombie on reload; new events will promote to active
```

This aligns with the persistent world design (GP-5): agents remain in the world as historical
entities (zombies), and new hook events from active Claude sessions promote them back to ACTIVE
via the normal inference path in `AgentInferenceEngine`.
