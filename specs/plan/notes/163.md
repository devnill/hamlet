# WI-163: Wire mcp_port setting to HTTP server and add /hamlet/health endpoint

## Context
Two independent issues:
1. `Settings` validates `mcp_port` (WI-112) but `MCPServer` ignores it — hardcoded to port 8080. Users cannot change the HTTP endpoint port.
2. `install.py` calls `http://localhost:8080/hamlet/health` but no `/hamlet/health` route exists — install always shows "MCP server not reachable" warning even when server is running.

## Changes

### src/hamlet/mcp_server/server.py
1. Add `port: int = 8080` parameter to `MCPServer.__init__`:
   ```python
   def __init__(self, ..., port: int = 8080):
       ...
       self._port = port
   ```
2. Replace hardcoded `8080` with `self._port` in the `web.TCPSite` or `runner.setup()` / `site.start()` call.
3. Add GET `/hamlet/health` route:
   ```python
   async def _handle_health(self, request) -> web.Response:
       return web.json_response({"status": "ok"})

   # In setup, add route:
   app.router.add_get("/hamlet/health", self._handle_health)
   ```

### src/hamlet/__main__.py (or daemon.py)
Pass `settings.mcp_port` when constructing MCPServer:
```python
mcp_server = MCPServer(..., port=settings.mcp_port)
```
