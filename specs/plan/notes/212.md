# WI-212 Implementation Notes: Fix legend overlay positioning

## Problem

`LegendOverlay` is yielded as the 4th child of `HamletApp.compose()` into a 3-row CSS grid.
Extra grid children that don't fit in defined cells get zero height — the widget is present
in the DOM but renders nothing when toggled visible.

## Fix

Two changes:

### 1. HamletApp — declare a layer for overlays

In `src/hamlet/tui/app.py`, add `LAYERS` to `HamletApp`:

```python
class HamletApp(App):
    LAYERS = ("default", "overlay")
    CSS = """..."""
```

### 2. LegendOverlay — position as floating overlay

In `src/hamlet/tui/legend.py`, update `DEFAULT_CSS`:

```python
DEFAULT_CSS = """
LegendOverlay {
    display: none;
    layer: overlay;
    position: absolute;
    top: 2;
    left: 2;
    width: 51;
    height: 22;
    background: $surface;
}
"""
```

The legend box drawn in `render()` is 49 chars wide and ~20 rows tall (count the lines).
Set `width` and `height` to match with a small margin. Adjust if the exact character count
differs.

If Textual's `position: absolute` behaves unexpectedly with the installed version (>=0.50.0),
an alternative is to set `dock: left` with `offset` — check Textual docs for the version
in use.

The same pattern should be applied to `HelpOverlay` in `src/hamlet/tui/help_overlay.py`
if it has the same positioning problem (out of scope for this work item unless it's trivial).
