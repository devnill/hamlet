# WI-155: Fix zombie frame formula in AnimationManager.get_frames()

## Context
WI-138 added get_frames() to AnimationManager but applies the spin formula (ticks // TICKS_PER_SPIN_FRAME % 4) to ALL agents, including zombie agents. Zombie agents should use the pulse formula (ticks // TICKS_PER_PULSE_FRAME % 2).

## Existing Code
- `src/hamlet/simulation/animation.py` — AnimationManager, get_frames() method
- `src/hamlet/simulation/animation.py` — TICKS_PER_SPIN_FRAME, TICKS_PER_PULSE_FRAME constants

## Fix
get_frames() needs to know which agents are zombies to apply the correct formula.

Option A: Add zombie_ids parameter:
```python
def get_frames(self, zombie_ids: set[str] = None) -> dict[str, int]:
    zombie_ids = zombie_ids or set()
    result = {}
    for agent_id, ticks in self._frames.items():
        if agent_id in zombie_ids:
            result[agent_id] = ticks // TICKS_PER_PULSE_FRAME % 2
        else:
            result[agent_id] = ticks // TICKS_PER_SPIN_FRAME % 4
    return result
```

Option B: Store agent state in AnimationManager and compute internally.

Option A is simpler — caller already knows zombie state from world_state.
