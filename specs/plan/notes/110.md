# Work Item 110: TESTER inference rules

## Problem
`AgentType.TESTER` can only be assigned manually — no inference rule classifies agents as TESTER from tool patterns. Agents running pytest are classified as EXECUTOR.

## Implementation
In `inference/engine.py`, in the `_infer_type` method, after applying TYPE_RULES:

```python
# Post-rule refinement: if EXECUTOR, check for test-running Bash commands
if inferred_type == AgentType.EXECUTOR:
    bash_events = [
        e for e in tool_window
        if e.tool_name == "Bash"
        and e.tool_input is not None
    ]
    test_keywords = ("pytest", "unittest", "test_", " test ", "run_tests", "tox", "nose")
    if bash_events:
        test_count = sum(
            1 for e in bash_events
            if any(kw in str(e.tool_input) for kw in test_keywords)
        )
        if test_count / len(bash_events) >= 0.5:
            inferred_type = AgentType.TESTER
```

## Notes
- `tool_window` is the sliding window of recent tool events used by _infer_type — check the actual variable name in engine.py
- `e.tool_input` may be a dict; `str(e.tool_input)` for keyword search is intentionally simple
- The 50% threshold is a soft heuristic; adjust if it produces false positives in practice
