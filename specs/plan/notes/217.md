# WI-217 Implementation Notes

## Context

`ExpansionManager.process_expansion` in `src/hamlet/simulation/expansion.py:66-86` runs on every simulation tick. The only gate is `check_village_expansion`, which returns a site whenever `len(agents) >= expansion_threshold`. Once a village reaches 20 agents this condition stays true permanently. On every subsequent tick the engine calls `create_road_between` (N lock acquisitions per road cell) and then `found_village`. Road cells are re-inserted into `_state.structures` and re-queued for SQLite on every tick — unbounded memory and write queue growth.

## Fix

### `src/hamlet/world_state/types.py`

Add `has_expanded: bool = False` to the `Village` dataclass. This field indicates that the village has already founded an outpost; no further expansion should be attempted.

### `src/hamlet/world_state/manager.py`

In `found_village`, after creating the new outpost village, set `has_expanded = True` on the **originating** village (the one that triggered the expansion). The originating village is identified by `project_id` and `village.id` (passed down from `process_expansion`).

The signature change may be needed: `found_village(self, originating_village_id: str, project_id: str, center: Position, name: str) -> Village` — or the caller passes the originating village and the method updates it. See expansion.py for how to thread this through.

Alternatively, the simpler fix: `process_expansion` sets `village.has_expanded = True` directly after `found_village` returns. Either approach is acceptable as long as the originating village's flag is set atomically with the founding call.

### `src/hamlet/simulation/expansion.py`

At the top of the per-village loop in `process_expansion`, add:
```python
if village.has_expanded:
    continue
```
This prevents re-running expansion for villages that have already founded an outpost.

## Note

The `has_expanded` flag is an in-memory-first design. If the daemon restarts, `Village` records are loaded from SQLite. The `has_expanded` field must also be persisted — check whether `Village` has a persistence schema in `src/hamlet/persistence/migrations.py` and `src/hamlet/persistence/loader.py`. If the `villages` table schema doesn't include `has_expanded`, add it. Check `src/hamlet/world_state/manager.py`'s `_village_to_dict` (or equivalent) method used for persistence writes.
