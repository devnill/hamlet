## server_url Fix

Current `find_server_url()` in all four hook scripts reads only from `~/.hamlet/config.json`:
```python
def find_server_url() -> str:
    default = "http://localhost:8080/hamlet/event"
    try:
        config_path = Path.home() / ".hamlet" / "config.json"
        if config_path.exists():
            data = json.loads(config_path.read_text())
            return data.get("server_url", default)
    except Exception:
        pass
    return default
```

`hamlet_init` (mcp/server.py) writes `server_url` to `<project>/.hamlet/config.json`. But the hooks never read this file for server_url.

Fix: Extend find_server_url() to also check the project config using the same upward traversal as find_config(). Precedence: project config server_url → global config server_url → hardcoded default.

Or more simply: merge into find_config() to return a triple (project_id, project_name, server_url) — this avoids duplicating the traversal. The traversal already happens in find_config(); server_url can be extracted from the same config file at the same time.
