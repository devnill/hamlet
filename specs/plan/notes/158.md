# WI-158: Fix hook scripts to read stdin and implement find_config()

## Context
All four hook scripts never read from stdin. They use `os.environ.get("CLAUDE_TOOL_NAME", "")` which is not a Claude Code environment variable and always returns empty string. They also use `os.environ.get("CLAUDE_PROJECT_ID", "unknown")` which is similarly not a real variable — all events arrive with `project_id="unknown"`.

Claude Code delivers hook data as JSON on stdin. For PreToolUse/PostToolUse, stdin contains `tool_name`, `tool_input`, `tool_response` (PostToolUse), `session_id`, etc.

## Root Cause (Gap-analyst G3, G4, Code-quality S3)
- G3: hooks never call `json.load(sys.stdin)`
- G4: hooks use fabricated env vars for project identity
- S3: tool_input and duration_ms never sent (so TESTER inference and work-unit scaling are dead)

## Changes

### All four hook scripts (pre_tool_use.py, post_tool_use.py, notification.py, stop.py)

Add `find_config()` function:
```python
def find_config() -> tuple[str, str]:
    """Traverse parent dirs looking for .hamlet/config.json.
    Returns (project_id, project_name).
    Falls back to hash of cwd if not found.
    """
    import hashlib
    cwd = Path(os.getcwd())
    for directory in [cwd, *cwd.parents]:
        config_path = directory / ".hamlet" / "config.json"
        if config_path.exists():
            try:
                cfg = json.loads(config_path.read_text())
                return cfg.get("project_id", ""), cfg.get("project_name", directory.name)
            except Exception:
                pass
    # Fallback: deterministic hash of cwd
    cwd_hash = hashlib.sha256(str(cwd).encode()).hexdigest()[:16]
    return f"project-{cwd_hash}", cwd.name
```

Read stdin at start of `main()`:
```python
hook_input = json.load(sys.stdin)
```

Extract fields from hook_input per hook type:
- PreToolUse: `hook_input.get("tool_name")`, `hook_input.get("tool_input")`
- PostToolUse: `hook_input.get("tool_name")`, `hook_input.get("tool_input")`, `hook_input.get("tool_response")`, `hook_input.get("duration_ms")`
- Notification: `hook_input.get("message")`, `hook_input.get("notification_type", "generic")`
- Stop: `hook_input.get("stop_reason", "unknown")`

Use `find_config()` instead of `os.environ.get("CLAUDE_PROJECT_ID", "unknown")`.

Fix error log to append:
```python
with log_file.open("a") as f:
    f.write(f"{error_msg}\n")
```
