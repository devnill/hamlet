## Content requirements

Keep under 150 lines. Focus on what an agent needs to avoid common mistakes.

### Running section
- Start daemon: `hamlet daemon` or `python -m hamlet daemon`
- Start viewer (connects to running daemon): `hamlet` or `hamlet view`
- They run as separate processes; start daemon first

### Testing section
- Command: `pytest` from the project root
- `asyncio_mode = "auto"` is set in pyproject.toml — no `@pytest.mark.asyncio` needed
- Test files are in `tests/`, named `test_{module}.py`
- Coverage: `pytest --tb=short`

### Architecture section (brief)
- Event pipeline order: Hook Script → HTTP POST → MCPServer → asyncio.Queue → EventProcessor → WorldStateManager + AgentInferenceEngine + PersistenceFacade
- All world state mutations acquire `self._lock` in WorldStateManager (asyncio.Lock, not threading.Lock — must be used with `async with`)
- Hook script pattern: `json.load(stdin)` → `os.chdir(cwd)` → `find_server_url()` → `find_config()` → `urllib.request.urlopen(POST)` → `sys.exit(0)` in finally

### Gotchas section
- **PreToolUse and PreCompact are synchronous hooks** (no `"async": true` in hooks.json) — they block Claude Code. All other hooks are async.
- **`sys.exit(0)` in all hook `finally` blocks** — tests must catch SystemExit or mock sys.exit
- **WorldStateManager re-entrant lock deadlock**: methods that acquire `self._lock` must not call other methods that also acquire it. The lock is `asyncio.Lock` not `threading.RLock` — it is not re-entrant.
- **Two separate MCP servers**: `mcp/server.py` is the plugin MCP server (stdio transport, exposes `hamlet_init` tool). `src/hamlet/mcp_server/server.py` is the application's HTTP event endpoint (aiohttp, receives hook events). They are different processes with different protocols.
- **`persistence/saver.py` is orphaned** — it exists (or did exist) but is not imported anywhere. Do not use it.
- **Hook scripts use stdlib only** — `urllib.request` not `aiohttp`, because hooks run outside the venv
