# WI-207: Fix _install idempotency guard and missing edge-case tests

## Context

Cycle 013 code-quality review (S1) identified that `_install`'s guard condition `if PLIST_PATH.exists() and _service_is_running()` fails to prevent a re-install when the plist exists but the service is stopped. In that state, the function overwrites the plist and calls `launchctl bootstrap` on an already-registered label, producing a confusing error.

## Fixes Required

### Fix 1 (S1): Expand `_install` idempotency guard

Replace lines 75-77 in `src/hamlet/cli/commands/service.py`:

```python
# BEFORE
if PLIST_PATH.exists() and _service_is_running():
    print("Service is already installed and running. Run `hamlet service uninstall` first.")
    return 0
```

With:

```python
# AFTER
if PLIST_PATH.exists():
    if _service_is_running():
        print("Service is already installed and running. Run `hamlet service uninstall` first.")
    else:
        print("Service is already installed but not running. Run `hamlet service start` to start it.")
    return 0
```

### Fix 3 (M4): Use subparser help for missing subcommand

In `src/hamlet/cli/__init__.py`, when registering the service subparser, store a reference to `service_parser` and configure it so that `service_command` can call `service_parser.print_help()` when no subcommand is provided.

OR: In `src/hamlet/cli/commands/service.py`, in `service_command`, change the fallback to call `print_help()` on the passed parser. However, `args` does not carry the parser reference directly.

Simpler approach: set `required=True` on the service subparsers object in `cli/__init__.py`:

```python
service_subparsers = service_parser.add_subparsers(dest="service_subcommand")
service_subparsers.required = True  # argparse prints help automatically if missing
```

This makes argparse itself reject the missing subcommand with a proper error message, eliminating the need for the manual dispatch fallback in `service_command`.
