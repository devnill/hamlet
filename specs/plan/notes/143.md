# WI-143: Fix mcp/start.sh path resolution

## Context
`mcp/start.sh` sets `SCRIPT_DIR="${CLAUDE_PLUGIN_ROOT}/mcp"` which breaks when `CLAUDE_PLUGIN_ROOT` is not set (expands to `/mcp`). The script should derive its location from `BASH_SOURCE[0]`.

Additionally, the `python3` fallback doesn't process PEP 723 inline script metadata and will fail at `from mcp.server import Server` unless `mcp` is pre-installed.

## Change to `mcp/start.sh`

```bash
#!/usr/bin/env bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
if command -v uv >/dev/null 2>&1; then
    exec uv run --script "${SCRIPT_DIR}/server.py"
else
    # python3 fallback: requires mcp package pre-installed in environment
    exec python3 "${SCRIPT_DIR}/server.py"
fi
```

## Notes
- `BASH_SOURCE[0]` gives the path to the script itself, even when sourced or called via symlink
- `$(cd ... && pwd)` resolves to the canonical absolute path
- The python3 fallback remains but is documented as requiring pre-installed mcp
- Read the current file first to preserve any existing logic
