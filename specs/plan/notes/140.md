# WI-140: Fix exit code propagation in main()

## Context
`main()` in `__main__.py` calls `asyncio.run(_run_viewer())` and discards the return value. `_run_viewer()` returns `1` on error (daemon unreachable, exception). The process always exits `0`.

## Existing Code
- `src/hamlet/__main__.py` — `main()` function, current no-args branch
- `_run_viewer()` returns int (0 or 1)

## Change

```python
def main() -> None:
    import sys
    args = sys.argv[1:]

    if not args:
        exit_code = asyncio.run(_run_viewer())
        sys.exit(exit_code)

    from hamlet.cli import main as cli_main
    sys.exit(cli_main(args))
```

## Notes
- Read current `main()` implementation before modifying
- `sys.exit(0)` on successful viewer run (or omit, since None → 0)
- `sys.exit(1)` on daemon unreachable or exception in viewer
- The `sys.exit(cli_main(args))` path is already correct — do not change it
