# WI-203: Update README.md Quick Start section

## Context

```markdown
## Quick Start

1. **Install hooks** (one-time setup):
   ```bash
   hamlet install
   ```

2. **Run Hamlet**:
   ```bash
   hamlet
   ```

3. **Use Claude Code normally** - your village will grow as you work!
```

This is missing:
- Project initialization step (`/hamlet:init` or `hamlet init`)
- The daemon as a separate step before the viewer

## Target

Replace the Quick Start section with a flow consistent with QUICKSTART.md:

```markdown
## Quick Start

1. **Initialize this project** (one-time per project):
   - In a Claude Code session: use the `/hamlet:init` skill
   - Or from the terminal: `hamlet init`

2. **Connect Claude Code** (one-time global setup):
   - Plugin users: hooks are registered automatically — skip this step
   - Others: `hamlet install`, then restart Claude Code

3. **Start the daemon**:
   ```bash
   hamlet daemon
   ```

4. **Open the viewer** (in a separate terminal):
   ```bash
   hamlet
   ```

5. **Use Claude Code normally** — your village grows as you work!
```
