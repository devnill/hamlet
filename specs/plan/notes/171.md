# WI-171 Notes: Add diagnostic for missing mcp package in start.sh

## Addresses
- Gap SG1: MCP server silently fails when uv absent and mcp not pre-installed

## File to modify
`/Users/dan/code/hamlet/mcp/start.sh`

## Current fallback
```sh
else
    # python3 fallback: requires mcp package pre-installed
    exec python3 "${SCRIPT_DIR}/server.py"
fi
```

## Desired behavior
Before executing server.py, check if mcp is importable. If not, emit a diagnostic to stderr and exit 1.

```sh
else
    # python3 fallback: requires mcp package pre-installed
    if ! python3 -c "import mcp" 2>/dev/null; then
        echo "hamlet: mcp package not found. Install uv (https://docs.astral.sh/uv/) or run: pip install mcp" >&2
        exit 1
    fi
    exec python3 "${SCRIPT_DIR}/server.py"
fi
```

## Constraints
- Script must remain POSIX sh compatible (#!/bin/sh is the current shebang)
- `python3 -c "import mcp"` is POSIX-compatible
- The diagnostic goes to stderr (>&2) so it appears in Claude Code's MCP server error log
- Exit 1 causes Claude Code to record the MCP server as failed to launch — better than silent ModuleNotFoundError
