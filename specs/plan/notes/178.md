# WI-178 Notes: Add error logging on malformed config JSON in hamlet_hook_utils.py

## Context
Gap analysis EC2 (significant): `find_config()` falls through to a phantom `project_id` when `.hamlet/config.json` contains malformed JSON. `except Exception: pass` swallows the error silently, routing hook events to a ghost village. EC1 (minor): same issue in `find_server_url()`.

The fix is to call `_log_error` inside each `except` block before continuing, matching the error-handling pattern already used in the hook scripts' `main()` functions.

## File to modify
`/Users/dan/code/hamlet/hooks/hamlet_hook_utils.py`

## Current find_config() (lines ~46-64)
```python
def find_config() -> tuple[str, str]:
    """Traverse from cwd up to root looking for .hamlet/config.json with project_id."""
    cwd = Path(os.getcwd())
    for directory in [cwd, *cwd.parents]:
        config_path = directory / ".hamlet" / "config.json"
        if config_path.exists():
            try:
                cfg = json.loads(config_path.read_text())
                pid = cfg.get("project_id", "")
                if pid:
                    return pid, cfg.get("project_name", directory.name)
            except Exception:
                pass
    return f"project-{_cwd_hash(cwd)}", cwd.name
```

    """Traverse from cwd up to root looking for .hamlet/config.json with project_id."""
    cwd = Path(os.getcwd())
    for directory in [cwd, *cwd.parents]:
        config_path = directory / ".hamlet" / "config.json"
        if config_path.exists():
            try:
                cfg = json.loads(config_path.read_text())
                pid = cfg.get("project_id", "")
                if pid:
                    return pid, cfg.get("project_name", directory.name)
            except Exception as exc:
                _log_error("find_config", exc)
