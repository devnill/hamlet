# Work Item 099: AgentType enum consolidation

## Problem
Two separate `AgentType` enums exist in `inference/types.py` (6 members, no PLANNER) and `world_state/types.py` (7 members, includes PLANNER). This causes runtime failures when converting between them. Additionally, `EXECUTOR` is missing from `tui/symbols.py` AGENT_COLORS, defaulting silently to white.

## Changes

### inference/types.py
- Add `PLANNER = "planner"` to the `AgentType` enum
- Add `AgentType.PLANNER` entry to `TYPE_COLORS` dict (use `"dark_green"` or match `world_state/types.py` color convention)

### tui/symbols.py
- Add `AgentType.EXECUTOR` entry to `AGENT_COLORS` dict (use `"cyan"` to match inference/types.py TYPE_COLORS)
- Verify all 7 members are covered

## Verification
After this work item, the string `.value` of every member of the inference `AgentType` must have a corresponding member in `world_state` `AgentType`. The existing conversion `WSAgentType(inferred.value)` in engine.py must not raise KeyError for any reachable value.
