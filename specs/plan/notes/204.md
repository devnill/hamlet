# WI-204: hamlet service subcommand group with launchd integration

## Context

No service management exists in the codebase. The daemon runs only as a foreground process via `hamlet daemon`. This work item adds a `hamlet service` subcommand group that wraps macOS launchd to manage the daemon as a user-level background service.

## Implementation Notes

### Plist location and label

```python
PLIST_LABEL = "com.hamlet.daemon"
PLIST_PATH = Path.home() / "Library" / "LaunchAgents" / f"{PLIST_LABEL}.plist"
```

### Finding the hamlet executable

Use `shutil.which("hamlet")` first. If not found, fall back to checking `Path(sys.executable).parent / "hamlet"` (handles venv installs where `which` may miss the venv bin). If neither works, print a clear error and exit:

```
Error: could not find hamlet executable in PATH.
Make sure hamlet is installed: pip install hamlet
```

### Plist template

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.hamlet.daemon</string>
    <key>ProgramArguments</key>
    <array>
        <string>{hamlet_executable}</string>
        <string>daemon</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardOutPath</key>
    <string>{log_path}</string>
    <key>StandardErrorPath</key>
    <string>{err_log_path}</string>
</dict>
</plist>
```

Where:
- `{hamlet_executable}` — absolute path from `shutil.which("hamlet")`
- `{log_path}` — `str(Path.home() / ".hamlet" / "daemon.log")`
- `{err_log_path}` — `str(Path.home() / ".hamlet" / "daemon.err.log")`

### launchctl commands

All launchctl calls use `subprocess.run`. Check return code; non-zero means error.

```python
import subprocess, sys

def _launchctl(args: list[str]) -> tuple[int, str]:
    result = subprocess.run(["launchctl"] + args, capture_output=True, text=True)
    return result.returncode, result.stderr or result.stdout

def _service_is_running() -> bool:
    """Check if the service is loaded in launchctl."""
    rc, _ = _launchctl(["list", PLIST_LABEL])
    return rc == 0
```

### Command implementations

**install:**
1. Check `sys.platform == "darwin"` — if not, print error and exit 1
2. Find hamlet executable
3. Ensure `~/.hamlet/` exists
4. Write plist to `PLIST_PATH`
5. `launchctl load PLIST_PATH`
6. Print success message

**uninstall:**
1. Check platform
2. If service running: `launchctl unload PLIST_PATH`
3. If `PLIST_PATH.exists()`: remove it
4. Print success message (no error if already absent)

**start:**
1. Check platform
2. If plist doesn't exist: print "Service not installed. Run `hamlet service install` first." exit 1
3. If already running: print "Service is already running." exit 0
4. `launchctl load PLIST_PATH`

**stop:**
1. Check platform
2. If not running: print "Service is not running." exit 0
3. `launchctl unload PLIST_PATH`

**restart:**
1. Call stop logic, then start logic

**status:**
1. Check platform
2. Check if plist exists → "Service installed: yes/no"
3. Check if running via `launchctl list` → "Daemon running: yes/no"
4. If running, optionally hit `http://localhost:{port}/hamlet/health` to confirm responsive
