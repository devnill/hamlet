# WI-240: TerrainConfig v2 — configuration parameters for multi-octave noise

## Context

The current `TerrainConfig` dataclass contains basic parameters for single-frequency Perlin noise generation. For realistic terrain with fractal Brownian motion (fBm) and domain warping, we need additional parameters.

## Implementation Notes

### Location
- `src/hamlet/world_state/terrain.py` — modify `TerrainConfig` dataclass

### Current State

```python
@dataclass
class TerrainConfig:
    seed: int | None = None
    world_size: int = 200
    water_frequency: float = 0.02
    mountain_frequency: float = 0.015
    forest_threshold: float = 0.4
    meadow_threshold: float = 0.6
    water_threshold: float = -0.3
    mountain_threshold: float = 0.7
```

### Required Additions

Add parameters for multi-octave noise (fBm):
- `octaves: int = 4` — number of noise layers to combine
- `lacunarity: float = 2.0` — frequency multiplier per octave (standard value)
- `persistence: float = 0.5` — amplitude decay per octave (standard value)

Add parameter for domain warping:
- `domain_warp_strength: float = 0.5` — strength of coordinate distortion

### Thresholds

The existing threshold parameters (`water_threshold`, `mountain_threshold`, `forest_threshold`, `meadow_threshold`) will be repurposed in WI-242 for the unified heightmap classification. The interpretation changes from "separate noise map threshold" to "heightmap elevation threshold" — but the values and parameter names can remain the same.

### Backward Compatibility

The default values for new parameters should produce reasonable terrain without requiring user configuration. The existing `TerrainConfig(seed=42)` instantiation pattern must continue to work.
