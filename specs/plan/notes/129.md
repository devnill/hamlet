# WI-129: Create hooks.json and shell wrappers

## Context
`hooks/hooks.json` tells Claude Code which hook events to intercept and which scripts to run. Shell wrapper scripts invoke the existing Python hook scripts using `${CLAUDE_PLUGIN_ROOT}` to locate them regardless of install path.

## Files to Create

### `hooks/hooks.json`
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/hooks/pre_tool_use.sh",
            "timeout": 5
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/hooks/post_tool_use.sh",
            "timeout": 5
          }
        ]
      }
    ],
    "Notification": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/hooks/notification.sh",
            "timeout": 5
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/hooks/stop.sh",
            "timeout": 5
          }
        ]
      }
    ]
  }
}
```

### `hooks/pre_tool_use.sh`
```bash
#!/usr/bin/env bash
exec "${CLAUDE_PLUGIN_ROOT}/hooks/pre_tool_use.py"
```

### `hooks/post_tool_use.sh`
```bash
#!/usr/bin/env bash
exec "${CLAUDE_PLUGIN_ROOT}/hooks/post_tool_use.py"
```

### `hooks/notification.sh`
```bash
#!/usr/bin/env bash
exec "${CLAUDE_PLUGIN_ROOT}/hooks/notification.sh"
```

Wait — note that `notification.sh` should call `notification.py`, not itself:
```bash
#!/usr/bin/env bash
exec "${CLAUDE_PLUGIN_ROOT}/hooks/notification.py"
```

### `hooks/stop.sh`
```bash
#!/usr/bin/env bash
exec "${CLAUDE_PLUGIN_ROOT}/hooks/stop.py"
```

## Notes
- All `.sh` files must be made executable (`chmod +x`). Create them with executable bit set.
- `exec` replaces the shell process, so stdin/stdout/stderr are inherited by the Python script — the hook's JSON payload on stdin flows through correctly.
- The existing Python hook scripts (`hooks/pre_tool_use.py`, etc.) already read from stdin and POST to the hamlet server. They fail silently if the server is not running (GP-7, already implemented).
- The timeout of 5 seconds matches the hook design intent (1-second HTTP timeout + overhead).
