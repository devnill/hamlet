# WI-134: Create HelpOverlay widget

## Context
New modal overlay showing keyboard controls, toggled by `?`. Follows the same pattern as `LegendOverlay` in `src/hamlet/tui/legend.py`.

## Existing Code to Reference
- `src/hamlet/tui/legend.py` — `LegendOverlay` class. Copy its structure: `reactive[bool]` for `visible`, `watch_visible`, `on_key`, `render`. Mimic the visual style.

## Implementation

### `src/hamlet/tui/help_overlay.py` (create)
```python
"""Help overlay widget — keyboard controls reference."""
from __future__ import annotations
from textual.reactive import reactive
from textual.widgets import Static


class HelpOverlay(Static):
    """Modal overlay showing keyboard controls. Toggled by ?."""

    visible: reactive[bool] = reactive(False)

    def watch_visible(self, visible: bool) -> None:
        self.display = visible

    def on_key(self, event) -> None:
        if event.key == "question_mark":
            self.visible = not self.visible
            event.stop()
        elif event.key == "escape" and self.visible:
            self.visible = False
            event.stop()

    def render(self) -> str:
        return (
            "┌─ Help ──────────────────────────────┐\n"
            "│  Arrow keys / hjkl — scroll          │\n"
            "│  f — toggle follow mode              │\n"
            "│  / — legend                          │\n"
            "│  ? — this help menu                  │\n"
            "│  q — quit                            │\n"
            "│                                      │\n"
            "│  Press ? to toggle  •  Esc to close  │\n"
            "└──────────────────────────────────────┘"
        )
```

## Notes
- `self.display = visible` hides/shows the widget using Textual's built-in `display` property, matching the LegendOverlay pattern.
- Initial `display` should be `False` (hidden). Add `DEFAULT_CSS = "HelpOverlay { display: none; }"` or set it in `watch_visible` initial call.
- The `render()` content uses `"/"` for legend (reflecting the WI-132 change). The text should match the actual bindings in app.py after WI-132 and WI-133.
- The widget handles `?` key internally (toggling itself), but `app.py` also needs to call `action_toggle_help` — both paths should work. The `event.stop()` prevents double-toggling.
