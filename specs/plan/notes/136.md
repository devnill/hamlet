# WI-136: Update default CLI behavior for no-subcommand case

## Context
Currently, `hamlet` with no subcommand dispatches to `parser.print_help()`. After the daemon/viewer split, `hamlet` with no arguments should launch viewer mode.

## Existing Code to Reference
- `src/hamlet/cli/__init__.py` — `main()` function. The no-subcommand branch is currently `parser.print_help(); return 0`.
- `src/hamlet/__main__.py` — viewer mode function created by WI-127 (`_run_viewer`).

## Changes to `src/hamlet/cli/__init__.py`

1. Register `view` subcommand:
```python
view_parser = subparsers.add_parser("view", help="Open hamlet TUI viewer (daemon must be running)")
view_parser.add_argument("--url", default="http://localhost:8080", help="Daemon URL")
view_parser.set_defaults(command="view")
```

2. In dispatch, change the no-subcommand branch:
```python
# Before:
if not parsed_args.command:
    parser.print_help()
    return 0

# After:
if not parsed_args.command or parsed_args.command == "view":
    import asyncio
    from hamlet.__main__ import _run_viewer
    url = getattr(parsed_args, "url", "http://localhost:8080")
    asyncio.run(_run_viewer(url))
    return 0
```

## Notes
- `hamlet --help` still works because `--help` is handled by argparse before subcommand dispatch.
- `hamlet view --url http://localhost:9090` allows connecting to a non-default daemon port.
- The `_run_viewer` function is defined in `__main__.py` by WI-127. Import it here to avoid duplicating the asyncio startup logic.
