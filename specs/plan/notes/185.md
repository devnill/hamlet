# WI-185 Notes: Plugin update hygiene

## Context
When both hamlet install and the plugin are active, hooks fire twice. install_command() should detect the plugin and skip hook installation.

## installed_plugins.json structure
Dict format: {"hamlet@marketplace": {"scope": "user", "installPath": "...", "version": "...", ...}}
Detection: any key containing "hamlet" with a non-empty installPath.

## Implementation
Add before hook installation section (after MCP server check and server_url write):
```python
def is_plugin_active() -> bool:
    path = Path.home() / ".claude" / "plugins" / "installed_plugins.json"
    try:
        data = json.loads(path.read_text(encoding="utf-8"))
        plugins = data.get("plugins", data)  # handle both wrapped and unwrapped
        for key in plugins:
            if "hamlet" in key.lower():
                entries = plugins[key] if isinstance(plugins[key], list) else [plugins[key]]
                if any(e.get("installPath") for e in entries):
                    return True
    except Exception:
        pass
    return False
```

The server_url write and MCP check still run before this detection. Hooks writing is skipped. Returns 0.
