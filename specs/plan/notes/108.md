# Work Item 108: Consolidate structure rules to single source of truth

## Problem
Two conflicting definitions of structure thresholds and material progressions:
- `world_state/manager.py`: `STAGE_THRESHOLDS` + `MATERIAL_STAGES` (used for create_structure)
- `simulation/structure_updater.py`: `STRUCTURE_RULES` with different threshold values

Example conflict: FORGE thresholds are [300, 1500, 3000] in manager vs [150, 750, 1500] in updater.

## Solution
Canonical source: `simulation/config.py` (where other simulation config lives).

New dict in `simulation/config.py`:
```python
STRUCTURE_RULES: dict[StructureType, dict] = {
    StructureType.HOUSE:     {"thresholds": [100, 500, 1000],   "materials": ["wood", "wood", "stone", "brick"]},
    StructureType.WORKSHOP:  {"thresholds": [150, 750, 1500],   "materials": ["wood", "wood", "stone", "brick"]},
    StructureType.LIBRARY:   {"thresholds": [200, 1000, 2000],  "materials": ["wood", "wood", "stone", "brick"]},
    StructureType.FORGE:     {"thresholds": [300, 1500, 3000],  "materials": ["wood", "wood", "stone", "brick"]},
    StructureType.TOWER:     {"thresholds": [500, 2500, 5000],  "materials": ["wood", "wood", "stone", "brick"]},
    StructureType.ROAD:      {"thresholds": [50, 200, 400],     "materials": ["wood", "wood", "stone", "brick"]},
    StructureType.WELL:      {"thresholds": [75, 300, 600],     "materials": ["wood", "wood", "stone", "brick"]},
}
```

Use world_state/manager.py values as canonical (they are higher, more conservative thresholds).

Then update both consumers to import from this location.

- Make sure to add `StructureType` to the imports and `STRUCTURE_RULES` to `__all__` in config.py
