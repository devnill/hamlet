# WI-164: Fix EXECUTOR agent type color to avoid error-color collision

## Context
Principle 3 states: "Color should indicate status (green=zombie/idle, white=active, red=error)." Red is reserved for error states. However, AgentType.EXECUTOR has base color "red" in TYPE_COLORS, meaning a healthy working EXECUTOR agent looks identical to an error state.

Additionally, legend.py renders EXECUTOR as green (the old color from before a prior refactor), but the actual color is "red" — creating a misleading legend entry.

## Changes

### src/hamlet/inference/types.py
Change EXECUTOR color from "red" to "cyan":
```python
AgentType.EXECUTOR: "cyan",
```

Verify no other agent type already uses "cyan". If RESEARCHER uses "cyan", pick a different unique color (e.g., "bright_magenta", "dark_orange", "bright_cyan").

### src/hamlet/tui/legend.py (or symbols.py if AGENT_COLORS exists there)
Update the EXECUTOR legend entry to display the correct color. Read legend.py to find the exact EXECUTOR line and update it to match the new color from TYPE_COLORS.

If symbols.py has an AGENT_COLORS dict that also maps EXECUTOR, update it there too.
