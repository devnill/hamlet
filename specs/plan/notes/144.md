# WI-144: Fix RemoteWorldState correctness issues

## Context
Multiple correctness issues in the viewer data layer:

1. `get_animation_frame` has wrong signature — async and takes no `agent_id` param
2. `get_event_log` uses `[-limit:]` (most recent) but server sends events in insertion order; should be `[:limit]` or clarified
3. `_refresh_remote_state` in app.py calls `self._world_state.refresh()` which WorldStateManager doesn't have; should call `self._remote_provider` instead or guard with hasattr
4. RemoteStateProvider methods (check_health, fetch_state, fetch_events) access `self._session` without null guard

## Existing Code
- `src/hamlet/tui/remote_world_state.py` — RemoteWorldState class
- `src/hamlet/tui/remote_state.py` — RemoteStateProvider class
- `src/hamlet/tui/app.py` — `_refresh_remote_state` method

## Changes

### `src/hamlet/tui/remote_world_state.py`

Fix `get_animation_frame`:
```python
def get_animation_frame(self, agent_id: str) -> int:
    return self._cached_state.get("animation_frames", {}).get(agent_id, 0)
```
(Remove `async`, add `agent_id` parameter)

Fix `get_event_log`:
```python
async def get_event_log(self, limit: int = 100) -> list:
    return self._cached_events[:limit]
```
(Change `[-limit:]` to `[:limit:]`)

Fix `_parse_agent` and `_parse_structure` to use `.get("id")` with fallback:
```python
id=d.get("id", ""),  # not d["id"]
```

### `src/hamlet/tui/remote_state.py`

Add null guard at top of each method:
```python
async def fetch_state(self) -> dict:
    if self._session is None:
        return {}
    async with self._session.get(...) as r:
        return await r.json()
```

### `src/hamlet/tui/app.py`

Fix `_refresh_remote_state`:
```python
async def _refresh_remote_state(self) -> None:
    if self._remote_provider is not None and hasattr(self._world_state, "refresh"):
        await self._world_state.refresh()
```
Or better: call refresh through the provider directly. Read the actual implementation first.
