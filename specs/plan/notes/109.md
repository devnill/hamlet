# Work Item 109: Persist Agent.project_id to SQLite

## Problem
`Agent.project_id` is maintained in memory by `WorldStateManager` but is never written to or read from SQLite. After an app restart, all agents have `project_id=""`, breaking the project-village relationship mapping.

## Changes

### persistence/migrations.py
Add migration key `2`:
```python
2: """
    ALTER TABLE agents ADD COLUMN project_id TEXT DEFAULT '';
    UPDATE schema_version SET version = 2 WHERE version = 1;
""",
```

Note: SQLite's `ALTER TABLE ADD COLUMN` cannot be inside the same transaction as `executescript`. Check the existing migration runner pattern for how to handle this.

### persistence/writer.py
In the agents INSERT OR REPLACE, add `project_id` to the column list and values:
```python
INSERT OR REPLACE INTO agents (id, session_id, village_id, project_id, ...)
VALUES (?, ?, ?, ?, ...)
```
Map from `data["project_id"]` or `data.get("project_id", "")`.

### persistence/loader.py
In `_load_agents`, add `project_id` to the SELECT column list and to the returned dict.

## Notes
- Existing databases at migration version 1 will have the column added with empty string default — no data loss
- The migration runner must handle the version check correctly; test with a fresh DB and an existing v1 DB
- Check if `Agent` dataclass's `project_id` field is already present (it should be, per world_state/types.py)
