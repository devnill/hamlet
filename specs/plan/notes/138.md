# WI-138: Fix animation_manager wiring and frame serialization

## Context
Two call sites construct `AnimationManager()` but never pass it to `MCPServer`, so `self._animation_manager` is always `None` and `/hamlet/state` always returns `animation_frames: {}`. Additionally, `serializers.py` accesses `_frames` directly (private attribute) and returns raw tick counts, not display frame indices.

## Existing Code
- `src/hamlet/__main__.py` — `animation_manager = AnimationManager()` at ~line 96, not passed to MCPServer at ~line 100
- `src/hamlet/cli/commands/daemon.py` — same omission at ~line 102-106
- `src/hamlet/mcp_server/serializers.py` — `dict(animation_manager._frames)` for animation_frames
- `src/hamlet/simulation/animation.py` — `AnimationManager._frames` stores tick counts; has `TICKS_PER_SPIN_FRAME` constant

## Changes

### `src/hamlet/__main__.py`
Change `MCPServer(world_state=world_state, port=settings.mcp_port)` to include `animation_manager=animation_manager`.

### `src/hamlet/cli/commands/daemon.py`
Same fix — pass `animation_manager=animation_manager` to `MCPServer(...)`.

### `src/hamlet/simulation/animation.py`
Add public method:
```python
def get_frames(self) -> dict[str, int]:
    """Return per-agent display frame indices (0 to N_FRAMES-1)."""
    return {
        agent_id: ticks // TICKS_PER_SPIN_FRAME % N_SPIN_FRAMES
        for agent_id, ticks in self._frames.items()
    }
```
Check actual constant names in the file before writing.

### `src/hamlet/mcp_server/serializers.py`
Change:
```python
dict(animation_manager._frames)
```
To:
```python
animation_manager.get_frames() if animation_manager else {}
```
