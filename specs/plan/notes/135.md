### action_toggle_help()
```python
def action_toggle_help(self) -> None:
    try:
        overlay = self.query_one(HelpOverlay)
        overlay.visible = not overlay.visible
    except Exception as exc:
        logger.debug("toggle_help: %s", exc)
```

## Notes
- The `?` binding in BINDINGS causes `action_toggle_help` to be called by Textual's action system when `?` is pressed. The `HelpOverlay.on_key` also handles `?` — both toggle the same `visible` reactive. This double-handling is acceptable (both set `visible = not visible` on the same key press on the same widget, but Textual's action system should call the app's `action_toggle_help` first via BINDINGS, and `on_key` in the widget also fires — net effect: one toggle). If double-toggle is observed in testing, remove the `on_key` handler from `HelpOverlay` and rely solely on `action_toggle_help`.
