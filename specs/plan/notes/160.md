# WI-160: Call get_or_create_project before session/agent creation in inference engine

## Context
`AgentInferenceEngine._handle_pre_tool_use` calls `get_or_create_session(event.session_id, event.project_id)` and `get_or_create_agent(...)` but never calls `get_or_create_project(event.project_id, ...)`.

The method `get_or_create_project` exists at `manager.py:134` and is never called from the inference engine. As a result:
- Multiple sessions for the same project_id can land in different placeholder villages
- After restart, agents have a project_id with no corresponding Project in memory
- "One village per codebase" requirement from interview is never met

## Fix

### src/hamlet/inference/engine.py — _handle_pre_tool_use

Add at the beginning of the method (before get_or_create_session):
```python
await self._world_state.get_or_create_project(event.project_id, event.project_name)
```

This call is idempotent — it returns the existing project if one already exists for that project_id.

- Read manager.py to verify get_or_create_project signature matches
