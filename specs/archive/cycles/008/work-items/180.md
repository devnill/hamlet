# WI-180 Notes: New hook scripts — system/observation events

## Context
v0.4.0 hook expansion. These 5 scripts cover system and observation events: PostToolUseFailure, UserPromptSubmit, PreCompact, PostCompact, StopFailure.

## stdin field locations (per Claude Code docs)
All fields at top level:
- PostToolUseFailure: session_id, cwd, hook_event_name, tool_name, tool_input, tool_use_id, error, is_interrupt
- UserPromptSubmit: session_id, cwd, hook_event_name, prompt
- PreCompact: session_id, cwd, hook_event_name (minimal payload)
- PostCompact: session_id, cwd, hook_event_name (minimal payload)
- StopFailure: session_id, cwd, hook_event_name, error (dict with type and reason)

## Notes
- stop_failure.py: error is a dict with type (rate_limit/authentication_failed/billing_error/etc.) and reason string
- pre_compact.py and post_compact.py: send base fields only (no event-specific data needed)
- Same pattern as WI-179 for all
