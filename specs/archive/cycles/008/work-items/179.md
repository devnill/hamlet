# WI-179 Notes: New hook scripts — agent/session lifecycle

## Context
v0.4.0 adds all available Claude Code hook types. These 6 scripts cover agent and session lifecycle events: SessionStart, SessionEnd, SubagentStart, SubagentStop, TeammateIdle, TaskCompleted.

## Pattern
Follow hooks/notification.py exactly. Key points:
- sys.path.insert(0, str(Path(__file__).parent)) before hamlet_hook_utils import
- Read stdin: hook_input = json.load(sys.stdin)
- session_id from hook_input.get("session_id", "")
- Event-specific fields from hook_input directly (not nested under "data")
- Per docs: SubagentStart/Stop have agent_id, agent_type at top level of stdin
- find_config() uses cwd — pass hook_input.get("cwd") or os.getcwd()
- POST with urllib.request, timeout=1, exit 0 in finally

## stdin field locations (per Claude Code docs)
All fields are at the top level of the stdin JSON object:
- SessionStart: session_id, cwd, hook_event_name, source
- SessionEnd: session_id, cwd, hook_event_name, reason
- SubagentStart: session_id, cwd, hook_event_name, agent_id, agent_type
- SubagentStop: session_id, cwd, hook_event_name, agent_id, agent_type, last_assistant_message
- TeammateIdle: session_id, cwd, hook_event_name, task_id, task_subject, teammate_name
- TaskCompleted: session_id, cwd, hook_event_name, task_id, task_subject, task_description, teammate_name
